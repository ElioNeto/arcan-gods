## Ciclo 03 — Combate

### ✨ Features
- **Damage Formulas**: `calculatePhysicalDamage`, `calculateMagicalDamage`, `calculateDefenseReduction`, `calculateHitRate`, `calculateCritical`, `calculateAttackDamage`, `calculateMagicDamage`, `calculateExperienceGain`, `calculateGoldDrop`
- **CombatSystem**: Servidor valida range (Manhattan ≤ 2), cooldown (500ms), alvo vivo. Aplica dano via fórmulas, gerencia kill com XP e Gold
- **ENTITY_DAMAGED packet**: Dados completos do ataque (dano, crítico, bloqueio, HP restante, killed, expGain, goldGain)
- **handlePlayerAttack**: De stub `NOT_IMPLEMENTED` para sistema funcional
- **Stat points**: `Player.levelUp()` agora concede 5 pontos de atributo por nível
- **XP multiplier**: Bônus de 1.5x para monstros 5+ níveis acima, penalidade de 0.5x para 5- abaixo

### 🐛 Bug Fixes
- Level up não concedia pontos de atributo (RF-024) — ✅ corrigido

### 🧪 Testes
- 28 testes para damage formulas
- 13 testes para CombatSystem
- **Total: 247 testes (20 suites)**

## Ciclo 02 — Movimento e Mundo

### ✨ Features
- **Tilemap types**: `ITileMap`, `ITileLayer`, `ICollisionGrid`, `IPortal`, `ISpawnPoint`
- **Collision types**: `CollisionResult`, `CollisionEvent`
- **Movement types**: `Waypoint`, `MovementPath`, `PathResult`
- **PLAYER_MOVE** agora usa `destX`/`destY`
- **TilemapLoader**: Parse de Tiled JSON
- **CollisionGrid**: Matriz walkable com bounds checking
- **MapManager**: Cache de mapas, fallback, portal detection
- **lorencia.json**: Mapa mock 50×40 tiles com paredes, obstáculos, 2 portais
- **Pathfinding A***: Manhattan heuristic, BinaryHeap, PathCache LRU com TTL
- **CollisionSystem**: Validação tile, wall-sliding, neighbor discovery
- **MovementSystem**: Movimento contínuo com acúmulo fracionário
- **Broadcast infra**: Multi-jogador no mesmo mapa
- **TilemapRenderer (cliente)**: Renderização do grid como tiles coloridos
- **MovementInterpolator (cliente)**: Interpolação suave entre waypoints

## Ciclo 01 — Fundação

### ✨ Features
- Monorepo npm workspaces (shared, server, client)
- TypeScript strict mode configurado
- ESLint + Vitest configurados
- **Shared**: Tipos de entidades, pacotes de rede, enums, constantes, schemas Zod, helpers
- **Server**: WebSocket (ws), heartbeat, rate limiting, graceful shutdown
- **GameEngine**: Tick loop 10Hz, respawn de monstros
- **World**: CRUD de players e monsters, world state
- **Player**: HP, MP, XP, level up, damage, healing
- **Monster**: Template-based, damage/defense, death/respawn
- **Auth**: Login/Register via WS (dev mode)
- **Client**: PixiJS 8 + Vite, game loop, NetworkManager, InputManager, Camera, AssetManager
- **MenuScreen**: Tela inicial com botão Connect
- **PlaceholderGraphics**: Entidades como quadrados coloridos
- **Chat**: Mensagens com validação Zod

- 13 testes shared
- 33 testes server
- 8 testes client
- **Total inicial: 73 testes**
