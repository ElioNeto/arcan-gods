## Novas Funcionalidades

### Fundação do Projeto

- **Monorepo Setup** (`#1`): Estrutura npm workspaces com `packages/shared`, `packages/server`, `packages/client`. TypeScript strict mode, ESLint, scripts `dev`/`build`/`lint`/`test` na raiz.
- **Shared Package** (`#8`): Tipos de entidades, enums, constantes de jogo/network, schemas de validação (Zod), funções utilitárias (`calculateLevel`, `xpForLevel`, `validateEmail`, `validateCharacterName`). 13 testes unitários.
- **Servidor Node.js + WebSocket** (`#2`): Servidor HTTP/WS na porta 3001, heartbeat (3s), rate limiting por conexão, roteamento de mensagens tipado, engine loop a 10 Hz, mundo com spawn de monstros (10), tratamento de graceful shutdown. 33 testes.
- **Cliente PixiJS + Vite** (`#3`): Game loop com phases (init → loading → menu → world), WebSocket client com reconexão automática (5 tentativas), input manager (mouse + teclado), câmera smooth follow, asset manager com placeholders canvas, tela de menu com gradiente + botão "Conectar", gráficos placeholders (player azul, monster vermelho, NPC verde). 8 testes.

### Autenticação (Dev Mode)

- Login/Register via WebSocket com auto-login para desenvolvimento
- Validação Zod nos payloads (LoginSchema, RegisterSchema)
- Token mock (`test-token-*`) para sessão
- Envio automático de `WORLD_STATE` após autenticação bem-sucedida

### Movimento

- Protocolo `PLAYER_MOVE` → `PLAYER_MOVED`
- Validação anti-teleporte no servidor (max 5 tiles/packet)
- Validação de bounds (0-255) via Zod `MoveSchema`
- Correção de posição no cliente quando movimento é rejeitado

### Chat

- Protocolo `PLAYER_CHAT` → `CHAT_MESSAGE`
- 4 canais: global, party, guild, whisper
- Validação server-side via `ChatSchema` (Zod)
- Eco no canal global com nome do jogador

## Mudanças na Arquitetura

- **Networking:** Substituição de Colysey por `ws` diretamente para simplificar o setup inicial (decisão do scoper)
- **Server-authoritative:** Movimento validado no servidor com anti-speedhack desde o primeiro ciclo
- **Auto-login:** Dev mode permite login sem banco de dados — qualquer email+senha válidos criam player automaticamente
- **Placeholders:** Gráficos gerados via Canvas API (círculos coloridos) — sem dependência de assets externos
