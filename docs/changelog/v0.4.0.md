## Ciclo 04 — Monster AI + HUD + Combat Feedback + Stamina + ChatSchema Tests

### ✨ Novas Funcionalidades

#### P0.1 — Monster AI FSM (#51)
- **5-state FSM**: IDLE → AGGRO → CHASE → ATTACK → RETURN, com transições completas e validação de leash
- **MonsterFSM**: Máquina de estados pura com `FSMUpdateResult`, testável em isolamento (24 testes)
- **MonsterAISystem**: Orquestração com stagger (1/3 dos monstros por tick) para distribuição de carga
- **Patrol Idle**: Monstros andam aleatoriamente no raio de patrulha a cada 3-5s no estado IDLE
- **Aggro**: Scan de players no `aggroRange`, seleciona o mais próximo, transita para AGGRO → CHASE
- **Attack**: Monstro ataca player via `CombatSystem.processMonsterAttack()` com range e cooldown
- **A* Pathfinding**: Chase e Return usam pathfinding A* com recálculo a cada 500ms
- **Broadcast**: `ENTITY_DAMAGED` transmitido para todo o mapa via `broadcastToMap()`
- **Respawn**: `Monster.respawn()` reseta estado da AI para IDLE
- **Performance**: Monitoramento de ticks lentos (>50ms) com log de warning
- **MonsterAIState packet**: Tipo `MONSTER_AI_STATE` definido em shared (não transmitido ativamente — dead code conhecido)

#### P0.2 — HUD Básico (#55)
- **HP Bar**: Barra vermelha 200×20px com fundo escuro
- **MP Bar**: Barra azul 200×20px com fundo escuro
- **XP Bar**: Barra dourada 200×12px com fundo escuro
- **Level Text**: "Lv. X" posicionado acima das barras
- **Name Text**: Nome do jogador exibido no topo
- **Safe Division**: `safeDiv()` previne divisão por zero em todos os casos (maxHp=0, maxMp=0, experienceToNext=0)
- **Resize Handler**: `resize()` mantém posição fixa top-left
- **Update em tempo real**: Atualiza via `ENTITY_DAMAGED`, `ENTITY_UPDATE`, `WORLD_STATE`, `AUTH_SUCCESS`
- **17 testes unitários**

#### P0.3 — Combat Feedback
- **DamageNumber**: Texto flutuante de dano com drift (-30px/s) e fade out (1.5s)
  - Dano normal: vermelho (#ff4444) 14px
  - Dano crítico: amarelo (#ffff44) 18px bold
  - Auto-remove via `isDead()` quando animação termina
- **EntityHealthBar**: Barra de HP 30×4px sobre entidades (Y_OFFSET = -8)
  - Atualização proporcional, clamp 0-1, suporte a maxHp=0
  - `destroy({ children: true })` para prevenir memory leak
  - Remove ao limpar entidade
- **CombatFeedbackManager**: Orquestrador
  - Cria damage numbers e health bars ao receber `ENTITY_DAMAGED`
  - Atualiza posição via `ENTITY_UPDATE`
  - Cleanup automático de damage numbers mortos
  - Filtro: apenas monstros (não player local)
- **32+ testes unitários**

#### P1.2 — Stamina (#57)
- `stamina`/`maxStamina` adicionados a `IPlayer` (shared)
- **Constantes**: `STAMINA_COST_PER_TILE = 1`, `STAMINA_REGEN_PER_TICK = 1`, `BASE_STAMINA = 100`
- **Consumo**: `MovementSystem.update()` consome 1 stamina por tile movido
- **Regeneração**: `GameEngine.tick()` regenera 1 stamina por tick quando parado
- **Clamping**: Stamina nunca abaixo de 0 nem acima de maxStamina
- **Incluído no toJSON()**: Enviado ao cliente via WORLD_STATE e AUTH_SUCCESS
- **4 testes unitários** (GameEngine)

#### P1.3 — ChatSchema Tests (#58)
- **11 testes** de validação cobrindo:
  - Mensagens válidas (1 char, 200 chars, "Hello!")
  - Mensagens inválidas (vazia, 201+ chars)
  - Channel enum (global, party, guild, whisper + inválido)
  - Case-sensitive channel ("GLOBAL" rejeitado)
  - Caracteres especiais, unicode, espaços, newlines
